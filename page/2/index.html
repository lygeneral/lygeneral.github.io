日本之旅-东京镰仓富士山京都大阪奈良

发表于 2019-01-30 更新于 2021-12-26 分类于 游记


日本一共有一都（东京都）一道（北海道）两府（大阪府和京都府）四十三县。本次长达9天的行程包含日本东京、镰仓、富士山、京都、大阪和奈良。在日本乘坐扶梯时发现不同地方的站位有所不同，在东京适应了靠左侧站立，到了大阪又变成了靠右站立。查了一下资料有以下三种可能：1）大阪万博会时，根据国际标准，在扶梯等公共场合都是“左侧通行”，便由此固定下来了这个模式；2）在大阪梅田第一次设置自动人行道的时候，用右手扶着扶手乘坐的人较多，于是就呼吁大家把左边空出来；3）在江户时代，居住着很多武士，为了不让刀鞘相碰，大家都习惯左侧通行。另一方，大阪的商人有着右手拿算盘和钱袋走路的习惯，所以遗传到现在，人们还是喜欢靠右行走。

交通

航班

可以在全日空ANA官网购买往返机票，越提前越便宜。

新干线

东京可以乘坐东海道、山阳新干线（800元左右）至京都，从右侧可以看到富士山。由于我们有富士山的行程，所以从东京到富士山停留一晚后，从富士山乘坐巴士去三岛，再从三岛乘坐东海道、山阳新干线（600元左右）到京都。

公交/地铁

ICOCA（西日本购买退卡）和SUICA（东日本购买退卡）在日本大部分区域均可以使用，若不考虑退卡没有什么区别。

如果行程中需要多次乘坐地铁，可以考虑东京/京都Metro1-3日券，票券有效期从第一次使用激活后开始计算使用时长，按照24小时制进行计时（大部分地铁运营时间：5:00-24:00）

住宿

airbnb预定民宿，订了两家民宿都比较坑，由于出行时间在樱花季，价格偏高而且住宿条件不怎么样。除非有认识的人住过或网上强烈推荐的民宿可供选择，否则不推荐。本来在airbub上预定了民宿，但是条件很差，便在东京新宿附近现找酒店，走到西铁Inn酒店里面询了价感觉不错就住下，不想折腾了，没想到非常干净舒适，比较推荐。

Agoda和Booking预定酒店/民宿，河口湖温泉酒店本来想预定富士豪景酒店的，但是富士豪景酒店没房了，只好在Agoda上预定了河口湖温泉新世纪酒店。新世纪酒店服务非常好，住宿条件也不错，价格合适的话比较推荐。我们在Booking上预定了大阪的比尤纳戈米套房度假屋，和照片一模一样，条件很好，离地铁站也很近，强烈推荐。

行程制定

虽然国内可以访问谷歌地图，但是不能制定行程，最好的是翻墙后访问国外的谷歌地图。首先要注册一个Google账号，在谷歌地图界面中点击左上角的菜单，选择[你的地点]，在[地图]中点击建立地图，就可以新建图层，添加酒店、景点信息，制定旅游线路。以下是我指定的日本旅游行程。

Hexo + Github Pages搭建个人博客网站

发表于 2019-01-04 更新于 2021-12-26 分类于 博客

一直想建立一个自己的网站，却因为各种事情给耽误了。随着学习的深入，我发现在信息爆炸的时代必须要经常总结才能提炼出对自己有用的东西，而且以后能够回顾。但是我又不喜欢受到各种博客平台的限制，就想着自己搭建一个博客网站，不仅可以记录自己的学习状态，而且可以向大家分享一些技术。于是花了几天时间搭建了个人的博客网站。

1 存储空间

搭建个人博客网站常采用云服务器/虚拟主机+WordPress和Github Pages+Hexo/jekyll这两种方案。文字、图片、视频等内容存储在云服务器/虚拟主机或者Github Pages上。

1.1 云服务器/虚拟主机

学生的话可以考虑阿里云和腾讯云，学生专享价10元/月，几乎打了一折。AWS提供了一年的免费主机，也值得考虑，可以免费试用一年。还有七牛云、老鹰主机和老薛主机等可供选择。

1.2 Github Pages

由于Github提供1GB的免费存储空间，单纯地存储一些文章和图片也已经够用了。同时为了激励自己去使用Github，我选择使用Github Pages搭建个人博客。

首先注册Github账号，然后新建仓库，和常规新建仓库的操作相同。要注意的是Repository name需要填写yourgithubname.github.io，其中yourgithubname和Github用户名一致。这样Github会默认新建的仓库配置为Github Pages，可以省去后续的配置。然后点击Create repository创建一个仓库。

至此，yourgithubname.github.io可以作为独立的网址进行访问了。只是现在还没有部署博客，不能正确的显示，后续通过Hexo把博客部署于Github Pages上，就可以看到完整的博客了。

3 域名

如果觉得yourgithubname.github.io比较low，可以向国内或国外的服务商申请自己的域名并把域名和Github Pages绑定在一起，就可以通过自己的域名访问博客了。如果存储空间购买国外的云服务器/虚拟主机或使用Github Pages的话，可以考虑从国外服务商申请域名，这样就不需要备案，省去一些事。

3.1 申请域名

通过国内域名服务商申请域名的话，可以去新网，万网（阿里云旗下）申请。

国外域名服务商就非常多了，有GoDaddy、namesilo、namecheap、name等等。很多都支持支付宝付款，购买也很方便。大家可以通过TLD-LIST和Domcomp这两个比价网站比较不同服务商域名的价格。

GoDaddy很受欢迎，许多人都推荐Godaddy，所以一开始我也是准备去GoDaddy申请域名。搜索我想要的域名后看到价格也很满意，就准备注册GoDaddy的账号了。注册GoDaddy的账号需要通过短信或者手机应用程序两步验证，我想通过短信验的方式注册手机却一直收不到短信，然后下载了两步验证工具Authy才注册成功。然后填入网上搜索的优惠码，一年只需要花几块钱，但是GoDaddy需要额外花60元购买隐私保护。而且听说GoDaddy的续费比较高，而namesilo送隐私保护并且续费也很合理，所以我后来选择去namesilo申请域名了。然后搜索它的优惠码并在箭头处输入提交，却发现通通都是减1美元，最后花了6美元申请了域名一年的期限。注意购买的时候选择WHOIS Privacy，可以保护隐私。

3.2 域名解析

域名解析可以使用申请域名的服务商提供的域名服务器解析域名，但namesilo是国外的域名服务器，博客基本是在国内进行访问。因此，我决定使用比较快而且稳定的第三方DNSPod进行域名解析。在以下使用namesilo和DNSPod解析域名的两种方法中选择一种即可。

3.2.1 namesilo解析域名

进入namesilo官网，点击右上角的Manage My Domains，出现申请的域名，然后点击箭头处的Manage DNS for this domain进入域名解析界面。

将所有默认的DNS记录删除，新建TARGET HOSTNAME为yourgithubname.github.io，HOSTNAME分别为@和www的两条CNAME记录。

然后进入域名，添加记录。和namesilo一样，添加记录值为yourgithubname.github.io，主机记录为@和www的两条CNAME记录，并不需要采取[ping yourgithubname.github.io]获取IP然后添加IP的A记录的方式。由于yourgithubname.github.io是国外的网站，因此等待全球DNS解析完成才能正常访问，需要耐心等待一段时间。

在学习工作中使用一些比较好用的软件，可以提高工作效率。

常用

Everything-文件搜索

基于名称实时定位文件和目录，进行文件搜索。搜索文件和文件夹的功能非常强大，比直接在资源管理器搜索快N倍。

Clover-Windonws Explorer资源管理器

增加多标签页，可以像浏览器一样可以打开多个不同路径的文件夹并随时切换查看。

具备书签栏，可以为经常浏览的文件夹新建书签。

Fences-桌面管理

把桌面的图标划分成多个区域进行分类整理，像栅栏一样布置,让桌面显得更加的整洁。

KeePass Password Safe-密码管理

Keepass包含强大的密码产生引擎与加密储存机制，提供安全的密码储存空间。不仅能生成随机的密码，而且可以记录日常使用的账号密码。

写论文

EndNote-文献管理

EndNote是一个参考文献和图书资料库，负责管理、存储以及查询文献及图书。基于EndNote引用文献的文章修改内容及格式后，会自动对文献顺序进行重排，起到事倍功半的效果。

写代码

Notepad++

Windows操作系统下的文本编辑器，适用于制作一般的纯文字说明文件，也非常适合编写计算机程序代码。

Beyond Compare-文本比较

对比两个文件夹或者文件，并将差异以颜色标示，有助于不同版本代码内容的比较。

产品工具

Axure RP

一个专业的快速原型设计工具，快速创建应用软件或Web网站的线框图、流程图、原型和规格说明文档。
